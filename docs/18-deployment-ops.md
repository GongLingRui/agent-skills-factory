## 部署拓扑

```
┌─────────────────────────────────────────┐
│              Nginx / LB                  │  ← 负载均衡 + SSL 终止
└──────────────┬──────────────────────────┘
               ↓
┌─────────────────────────────────────────┐
│         API Gateway (3 实例)             │  ← 无状态，可水平扩展
└──────────────┬──────────────────────────┘
               ↓
┌─────────────────────────────────────────┐
│      Agent App Factory Core (3 实例)     │  ← 核心业务逻辑
│  ┌─────────┐ ┌─────────┐ ┌──────────┐  │
│  │ 注册中心 │ │Compiler │ │ Runner   │  │
│  └─────────┘ └─────────┘ └──────────┘  │
└──────────────┬──────────────────────────┘
               ↓
┌─────────────────────────────────────────┐
│          Redis（缓存 + 队列）             │  ← 1 主 1 从
│          PostgreSQL（持久化）             │  ← 1 主 1 从
│          MinIO / S3（Skill 包存储）       │  ← 对象存储
└─────────────────────────────────────────┘
```

## 组件部署要求

| 组件 | 实例数 | CPU | 内存 | 存储 | 说明 |
|------|--------|-----|------|------|------|
| Nginx | 2 | 2 核 | 4G | 20G | 负载均衡 |
| API Gateway | 3 | 2 核 | 4G | 20G | 无状态 |
| Core 服务 | 3 | 4 核 | 8G | 50G | 主要业务逻辑 |
| 文档解析 Worker | 2 | 4 核 | 8G | 100G | 独立部署 |
| 脚本 Worker | 2 | 2 核 | 4G | 50G | 沙箱环境 |
| Redis | 2 | 2 核 | 8G | 50G | 主从 |
| PostgreSQL | 2 | 4 核 | 16G | 200G | 主从 |
| MinIO | 2 | 2 核 | 4G | 500G | 对象存储 |

### 容量规划（支持 1000 DAU）

| 资源 | 估算依据 | 总量 |
|------|---------|------|
| **Core 服务** | 单实例 4C8G 支持 ~25 并发；1000 DAU 峰值并发 ~100 | 4 实例（冗余 1 个） |
| **API Gateway** | 无状态，按 Core 的 1.5 倍 | 3 实例 |
| **Redis** | 数据量 ~155MB，连接数瓶颈；8G 内存绰绰有余 | 1 主 1 从 |
| **PostgreSQL** | 审计日志每日 ~50MB，90 天 ≈ 4.5GB；其他元数据 < 1GB | 主 200G + 从 200G |
| **MinIO** | Skill 包 1000 个 × 平均 1MB = 1GB；temp 桶 24h 清理 | 100G |
| **文档解析 Worker** | 峰值 20 并发解析，单文件 10-30s | 4 实例 |
| **出口带宽** | SSE 流式：单会话 ~5KB/s；100 并发 ≈ 5Mbps | 100Mbps（含余量） |

**扩展到 10000 DAU**：Core 服务扩至 10 实例、Worker 扩至 10 实例、Redis 升配至 16G、PostgreSQL 升配至 8C32G。

## 降级运维手册

### 手动降级开关

运维人员可通过管理后台或 API 强制触发降级：

```bash
# 强制启用第 3 级降级（切到小模型）
curl -X POST /admin/degradation/level \
  -H "Authorization: Bearer <admin-token>" \
  -d '{"level": 3, "reason": "模型集群维护", "duration_minutes": 60}'

# 强制恢复
curl -X POST /admin/degradation/recover \
  -H "Authorization: Bearer <admin-token>"

# 屏蔽某个 Agent
curl -X POST /admin/agents/contract-review-agent/disable \
  -H "Authorization: Bearer <admin-token>" \
  -d '{"reason": "法务准确性问题", "duration_minutes": 30}'
```

### 降级级别对照

| 级别 | 动作 | 影响 |
|------|------|------|
| 1 | 跳过 rerank | 检索质量轻微下降 |
| 2 | 检索 top_k 从 20 降到 5 | 检索召回率下降 |
| 3 | 切到小模型 | 回答质量可能下降 |
| 4 | 限制 max_turns 到 3 | 长对话体验下降 |
| 5 | 关闭非必要工具 | 部分功能不可用 |
| 6 | 长文档转异步 | 大文件处理延迟增加 |

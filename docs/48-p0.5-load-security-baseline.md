## 压测与容量基线

| 层级 | 说明 |
|------|------|
| **文档目标** | [docs/27-testing-strategy.md](27-testing-strategy.md) §性能测试（网关 QPS、模型 P99 等）与 [docs/18-deployment-ops.md](18-deployment-ops.md) 组件容量表适用于 **staging / 专用压测环境**。 |
| **本仓库自动化** | `backend/tests/unit/test_health_load_smoke.py`：对 ASGI 栈并发请求 `/health`，断言全部成功且 P99 延迟低于宽松阈值（验证异步栈无基础性坍塌）。 |
| **可选脚本** | `backend/scripts/benchmark_health_smoke.py`：对**已启动**的 HTTP 服务采样 QPS/延迟（本地或预发）；不参与 CI。 |

完整 Locust/k6、chat 链路压测仍在预发环境按 docs/27 执行并留存报告。

## 安全加固与渗透复测

| 层级 | 说明 |
|------|------|
| **威胁与合规** | 以 [docs/45-security-architecture.md](45-security-architecture.md) STRIDE 与合规表为准。 |
